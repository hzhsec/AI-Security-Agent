# 2026-03-20 工作日志

## 问题修复：AI 对话界面显示 `<think>` 标签原文

- 现象：MiniMax-M1 等推理模型在 AI 对话助手里返回带 `<think>...</think>` 的思维链，前端直接把原文展示出来，用户看到大段原始推理内容
- 修复位置：index.html 的 `appendChatMsg()` 函数和 `sendChat()` 流式输出部分

### 改动内容

**① 新增 `stripThinkTags(text)` 工具函数**
- 用正则 `/<think>[\s\S]*?<\/think>/gi` 过滤所有思维链内容
- 在渲染 assistant 消息前调用，最终显示的内容不含思维链

**② 流式输出实时过滤**
- `sendChat()` 里打字效果阶段，每次 token 更新都调用 `stripThinkTags` 过滤显示内容
- 流结束后 `appendChatMsg` 二次渲染时也会过滤

**③ 推荐任务指令正则扩展**
- 原正则只匹配 `**✅ 推荐任务指令：**` 严格格式
- 新正则兼容：有无 `**` 加粗、不同分隔符（`：` `:`）、bash/shell/sh 代码块语言标记
- 正则：`/(?:\*{0,2}✅\s*推荐任务指令[：:\s]*\*{0,2})\s*\`\`\`(?:bash|shell|sh)?\s*([\s\S]*?)\`\`\`/i`
